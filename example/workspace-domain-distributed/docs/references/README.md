# Workspace References

This directory contains long-lived, non-authoritative workspace references.

Documents in this directory are auxiliary reference materials to support developers and analysts. They are explicitly **non-authoritative** and subject to the following rules:

1. **No Override Power:** They must not override the master `00_Documentation_Blueprint.md` or any domain specifications (e.g., BRD, PRD, User Stories).
2. **No Source-of-Truth Requirements:** They must not become the source of truth for product or system requirements.
3. **No Temporary Content:** Temporary files, scratch pads, or draft notes must be kept outside of this directory.
4. **Allowed Content:**
   - Long-lived onboarding guides, environmental configurations, or workspace setup instructions.
   - Hardware specifications, third-party API reference documents, or external protocol descriptions.
   - Policy documents, compliance standards, or regulatory definitions.
   - Legacy system architectures or legacy workflow diagrams.
